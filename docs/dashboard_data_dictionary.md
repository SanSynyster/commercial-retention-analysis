# Dashboard Data Dictionary

## Purpose

This document describes the dashboard-ready datasets generated by the notebooks. It is intended to keep Tableau or BI dashboard calculations consistent with the project's Python, SQL, and report outputs.

## Recommended Dashboard Pages

| Dashboard Page | Primary Datasets |
| --- | --- |
| Executive Overview | `data_audit_summary.csv`, `customer_lifecycle_revenue_split.csv`, `customer_revenue_deciles.csv` |
| Revenue and Retention | `customer_metrics.parquet`, `customer_lifecycle_revenue_split.csv` |
| Cohort Retention | `retention_matrix.csv` or SQL long-form cohort output |
| Customer Segments | `customer_segment_summary.csv`, `customer_segments.csv` |
| Predictive ML | `second_purchase_model_metrics.csv`, `second_purchase_propensity_lift.csv`, `second_purchase_propensity_bands.csv`, `lapse_risk_model_metrics.csv`, `lapse_risk_lift.csv` |
| CRM Priority Actions | `crm_priority_summary.csv`, `crm_customer_priority_scores.csv` |

## `data_audit_summary.csv`

Key columns:

| Column | Description |
| --- | --- |

- `net_transactional_revenue`

## Formatting Guidance

Use these display conventions:

- Percentages: one decimal place.
- Currency: compact GBP format, such as `GBP 8.9M`.
- Counts: comma-separated whole numbers.
- Model scores: two or three decimal places.
- Risk and priority groups: keep original labels for consistency with reports.
